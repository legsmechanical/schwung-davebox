# SEQ8 Session Summary — 2026-04-28

Features added and changed this session, described from a user perspective.

---

## 1. Drum NOTE FX — Lane Note Control (K7 + K8)

On a drum track, the NOTE FX bank (Shift + pad 94) now has two active controls in the bottom-right corner, displayed as a boxed region distinct from the real-time NOTE FX params above:

- **K7 (Oct)** — coarse control: moves the lane's MIDI note up or down one octave (±12 semitones) per detent
- **K8 (Note)** — fine control: moves the lane's MIDI note up or down one semitone per detent

Both knobs clamp at 0–127 and do not cycle. The display shows the note name and MIDI number (e.g. `C3 48`) centered in the box, with "Oct" and "Note" labels above. The box inverts when either knob is touched.

Lane note changes are now **persistent** — they survive Shift+Back saves and reloads.

---

## 2. Drum SEQ Bank — Nudge/Clock Shift Speed Fix

Turning K2 (clock shift) or K3 (nudge) on the drum DRUM SEQ bank is now responsive. Previously each detent triggered a full resync of all 32 lanes (64 IPC calls), causing audible lag. Now only the active lane's steps are re-read after each turn.

---

## 3. Melodic Track View Idle — Second Info Row

The melodic track view idle screen now shows a second row of information directly below the Knob label:

- **Left**: `Oct: +X` — the current pad octave shift for the active track (e.g. `Oct: +0`, `Oct: -2`)
- **Right**: key and scale name, right-aligned to the screen edge (e.g. `A Minor`, `C# Pent Major`)

Scale names are displayed in full except for four abbreviated forms: **H Minor** (Harmonic Minor), **M Minor** (Melodic Minor), **Pent Major** (Pentatonic Major), **Pent Minor** (Pentatonic Minor).

When **Scale Aware** is on (Global Menu → Scale Aware), a 1px underline appears beneath the key+scale text. The underline disappears when Scale Aware is off.

The octave info updates live as you press Up/Down to shift octave — no separate overlay popup.

---

## 4. Octave Overlay Removed

The ~1 second full-screen "Octave: +2" overlay that appeared after pressing Up/Down is gone. The octave value is now always visible in the idle screen second row (see above), so the overlay was redundant.

---

## 5. OLED Bank Header Format

All bank headers now use `[ LABEL ]` format with a space inside the brackets:

- `[ NOTE FX ]`, `[ DRUM SEQ ]`, `[ NOTE/NOTEFX ]`, `[ HARMZ ]`, etc.

This applies to both the bank overview grid and the `Knob:` idle line.

---

## 6. Knob Label Position

The `Knob: [ BANK ]` label is now at the very top of the OLED (y=0) on all track views and modes. Previously it was one row down.

On the drum track idle view, the `Bank: A  Pad: C3 (48)` info line moved up one row to sit directly under the Knob label.

---

## 7. Track Number Row — Even Distribution + Active Box

Track numbers 1–8 are now evenly distributed across the full 128px OLED width (16px per slot, character centered in each slot). Previously they were left-clustered in 12px spacing.

Clip letters on the row below are aligned to the same x positions, so each letter sits directly under its track number.

### Active track indicator
The active track number now has a **box** drawn around it (1px border with 1px padding around the character). No more underline.

### Mute indicator  
Muted tracks show a **filled box** (inverted — white background, black number) for all tracks, active or not.

### Solo indicator
Soloed tracks **blink the number** (the character appears and disappears). No box involvement for solo. If the active track is soloed, the number blinks inside the active-track outline box.

## Session 2 — Bug fixes and tweaks (2026-04-28 continued)

### Drum pad velocity sensitivity (fix)
Lane pads (left 4 cols) now use `effectiveVelocity(d2)` instead of fixed zone velocity. Velocity-sensitive playing works correctly.

### 16-level pad recording (fix)
Tapping a velocity zone pad while recording armed now calls `drum_record_note_on` at the zone velocity. Previously only lane pads triggered recording.

### Per-lane mute/solo OLED revamp
Replaced per-lane M/S character markers with centered full-word text: **"MUTED"** (flashing, /50 tick rate) and **"SOLOED"** (solid, white-on-black inverted). Both centered in second row of drum idle OLED.

### Drum step deactivation bug (fix — 2 parts)
**Symptom**: tapping an active drum step caused LED to go off but note kept playing.

**Root cause**: `note_step()` uses a half-open midpoint interval `[-tps/2, +tps/2)`. When drum K3 nudge hit exactly `-tps/2` (= -12 for default tps=24), JS fired reassign — moving step data to S-1 with `new_offset = +tps/2`. Since `+12` is outside the half-open interval, `note_step()` assigned the note back to step S. Data at S-1, display at S → `_clear` for S was a no-op.

**Fix 1** (JS `ui.js`): Change drum negative reassign threshold from `<= -tpsMid` to `< -tpsMid`. At nudge=−12 no reassign fires; note stays at step S with offset=−12, which `note_step` correctly maps to S. Matches melodic mode's asymmetric threshold (effectively nudge≤−13 for tps=24).

**Fix 2** (DSP `seq8.c`): Change `tN_lL_steps` get_param from iterating `notes[]` via `note_step()` to reading `steps[s]`/`step_note_count[s]` directly (step-centric). This ensures the drum step display always matches the physical step indices that `_clear`/`_toggle` operate on, regardless of tick offsets.
